# 为什么这样学

Next.js 的难点不是“会不会写页面”，而是你需要同时理解三套东西：

- React Server Components 的组件边界
- Next.js App Router 的路由和渲染流程
- 多层缓存、失效和预取策略

如果只看 API，很容易记住很多零散规则，却不知道项目里应该怎么选。这个教程采用“心智模型 + 小实验 + 最小实战”的方式。

## 学习原则

1. 先问“数据新鲜度”，再选渲染模式。
2. 先默认 Server Component，再把交互部分下沉为 Client Component。
3. 先用框架默认缓存，再在明确需要时声明 `use cache`、`cacheLife` 和 `cacheTag`。
4. 先让失效路径可解释，再考虑性能优化。
5. 每个章节都保留能运行或能验证的代码。

## 你不需要一开始掌握全部 API

渲染和缓存的 API 很多，但日常开发最常遇到的是这几类：

- 页面是否可静态化
- 数据是否需要每次请求都新鲜
- 是否可以接受短时间陈旧
- 用户操作后如何立刻看到新数据
- 是否需要对某一类数据统一失效

本教程围绕这些真实问题组织，而不是按 API 字母表排序。
